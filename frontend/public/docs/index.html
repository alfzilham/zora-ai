Introduction to ZORA AI

ZORA is a SuperIntelligence Autonomous AI platform that orchestrates multiple AI models to handle user requests. Users interact with a single unified identity — ZORA — while the system routes each request to the most capable model available.

What makes ZORA different? Unlike single-model AI assistants, ZORA silently delegates every request to the best-suited model — Qwen for code, Gemini for research, Minimax for creative writing — all under one unified identity.

What is ZORA?

ZORA is an AI orchestration layer built on FastAPI (Python) with a vanilla HTML/CSS/JS frontend. It classifies user intent via Groq, then routes to the optimal model from its pool of NVIDIA, Google, and Groq-hosted models.

Core Features

🧠 AI Orchestration
Automatically routes to the best model per task type.
💬 Multi-turn Chat
Persistent conversations with user memory injection.
🔒 Incognito Mode
Chat privately with zero history or data saved.
⚡ Extended Thinking
Deep reasoning layer before generating answers.
🌐 11 Languages
EN, ID, FR, DE, JA, IT, KO, PT, ES, ZH + more.
🔬 Zora Labs
Specialized tools: Code, Design, Image, Research.

Quickstart

Get ZORA AI running locally in under 5 minutes.

Prerequisites

Make sure you have the following installed before proceeding:

1
Python 3.11+
Required for the FastAPI backend and all AI service integrations.
2
Node.js 18+
Needed for the live-server dev environment and Vercel CLI.
3
PostgreSQL (via Neon)
Production database. Get a free connection string at neon.tech.

Setup

bash 
# 1. Clone the repo
git clone https://github.com/USERNAME/zora-ai.git
cd zora-ai

# 2. Copy environment file
cp .env.example backend/.env

# 3. Install backend dependencies
cd backend && pip install -r requirements.txt

# 4. Start the backend
uvicorn app.main:app --reload --port 8000

# 5. Serve the frontend (new terminal)
npm install -g live-server
live-server frontend --port=3000
Verify it's running: Open http://127.0.0.1:8000/docs for the Swagger UI and http://localhost:3000 for the frontend.

Architecture

ZORA is built as a modular orchestration platform. The frontend communicates exclusively with the FastAPI backend, which handles all AI routing, authentication, and database operations.

Tech Stack

Layer Technology Purpose
Frontend HTML, CSS, JavaScript (Vanilla) UI, no frameworks
Backend Python + FastAPI API, AI routing, auth
Database PostgreSQL (Neon) Users, chats, memory
Deploy Vercel Serverless hosting
Auth JWT + Google OAuth Session management
Payment Xendit Developer withdrawals

Request Flow

User Message
Intent Detector (Groq)
Returns JSON routing decision
Orchestrator
pre_think = true → Groq reasons first
Primary Model (NVIDIA / Gemini / Groq)
Fallback key on rate limit
Response as "ZORA"

Routing Logic
AI Models

Routing Logic

The Intent Detector analyzes every message and returns a JSON routing decision. This happens invisibly — users only ever see ZORA.

Routing Rules

Condition Routed To Pre-think
Code, programming, debug Qwen —
Math, logic, data analysis Deepseek Optional
Long document, PDF, large text Kimi —
Creative writing, story, ads Minimax —
Research, news, facts, citations Gemini —
Chinese language, multilingual GLM —
Complex multi-step reasoning Groq → Best Model ✓ Always
General chat, everyday Q&A Nemotron —
Any model rate limited Fallback Key —

Classifier Response Format

json 
{
  "primary_model": "qwen",
  "pre_think": false,
  "confidence": 0.97,
  "reason": "Code debugging task."
}

Chat System
Features

Chat System

The ZORA chat system supports streaming responses, multi-turn context, incognito mode, extended thinking, and conversation history management.

Chat Modes

Standard Chat
Full history saved to PostgreSQL. AI memory injected into system prompt. Conversations auto-titled from the first message.
Incognito Mode
No history saved. No memory injected. No user data stored during session. Routes to the same model pool.
Extended Thinking
Activates the Groq pre-thinking layer. Groq reasons through the problem first, then passes enriched context to the primary model.

Environment Variables

Copy .env.example to backend/.env and fill in all required values before running the server.

.env 
# Database
DATABASE_URL=postgresql://...

# JWT
JWT_SECRET=your_secret_here
JWT_ALGORITHM=HS256
JWT_EXPIRE_MINUTES=1440

# Google OAuth
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=

# NVIDIA — one key per model
NVIDIA_API_KEY_NEMOTRON=
NVIDIA_API_KEY_DEEPSEEK=
NVIDIA_API_KEY_QWEN=
NVIDIA_API_KEY_KIMI=
NVIDIA_API_KEY_MINIMAX=
NVIDIA_API_KEY_GLM=
NVIDIA_API_KEY_GEMMA=
NVIDIA_API_KEY_MISTRAL=
NVIDIA_API_KEY_FALLBACK=

# AI Services
GROQ_API_KEY=
GEMINI_API_KEY=

# Payment
XENDIT_SECRET_KEY=
XENDIT_WEBHOOK_TOKEN=

# App
APP_URL=http://localhost:3000
BACKEND_URL=http://localhost:8000
ENV=development

Deployment

Deploy to Vercel

ZORA deploys as a hybrid — FastAPI as Vercel Serverless Functions, frontend as static files. The vercel.json routes /api/* to FastAPI and everything else to the static frontend.

bash 
# Install Vercel CLI
npm install -g vercel

# Deploy to production
vercel --prod
Important: Add all environment variables in Vercel Dashboard → Settings → Environment Variables before deploying. The app will fail to start without DATABASE_URL and JWT_SECRET.
